## 3. 数据集计划

### 3.1 主数据集

本项目主数据集固定为 `BloodMNIST`。

选择原因：

- 属于医学图像领域，满足 domain-specific 要求
- 数据规模适中，适合完成多组迁移学习实验
- 自带标准划分，便于复现与公平比较
- 计算成本低于很多大型医学影像数据集

### 3.2 数据使用策略

直接使用 `BloodMNIST` 官方的：

- `train`
- `val`
- `test`

在做数据效率实验时：

- 只对子训练集做分层抽样
- `val` 和 `test` 保持不变

## 5. 架构对比计划

README 要求至少比较 3 个不同家族的预训练模型。考虑到你已经完成了 `ViT-B/16`，推荐补齐如下组合：

- `ResNet-50`
- `EfficientNet-B0` 或 `ConvNeXt-Tiny`
- `ViT-B/16`

推荐原因：

- `ResNet-50` 代表经典 CNN
- `EfficientNet-B0` 或 `ConvNeXt-Tiny` 代表更现代的卷积网络
- `ViT-B/16` 代表 Transformer 架构

每个模型都需要汇报：

- Test accuracy
- Macro F1
- 参数量
- 模型大小（MB）
- 推理延迟（ms/image, GPU）

当前建议：

- 保留 `ViT-B/16` 作为已完成结果
- 新增实现 `ResNet-50`
- 再新增实现 `EfficientNet-B0` 或 `ConvNeXt-Tiny`
